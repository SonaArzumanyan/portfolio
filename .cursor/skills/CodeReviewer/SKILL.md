# Code Reviewer Skill

This skill analyzes frontend code and provides structured feedback. It focuses on identifying issues, explaining problems, and suggesting improvements without rewriting the code.

---

## When to Use

- Use when:
  - User types "review"
  - User asks for feedback on code
  - User wants to understand issues or bugs
  - User wants improvement suggestions

---

## Instructions

### Step 1 — Analyze Code

Carefully inspect the provided code and identify:

- Bugs or potential errors
- Bad practices
- Unclear or risky logic
- Performance concerns

---

### Step 2 — Debugging Insight

If issues are found:

- Explain what is wrong
- Explain why it happens
- Describe possible consequences

---

### Step 3 — Suggestions (NO fixing)

- Suggest how to fix the issue
- Describe better approaches
- Recommend best practices

⚠️ Do NOT rewrite the full code  
⚠️ Do NOT provide a full fixed version unless explicitly asked  

---

### Step 4 — Output Format (STRICT)

## Issues
- ...

## Explanation
- ...

## Suggestions
- ...
